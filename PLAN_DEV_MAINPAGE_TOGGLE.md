# План: ліва dev-кнопка у режимі вправ (main-page)

Дата: 2026-02-28
Статус: узгоджено перед реалізацією

## Мета
Додати в `main-page` (режим проходження вправ) ліву кнопку розгортання dev-панелі з таким самим функціоналом, як у хедері сторінки вправ/налаштувань.

## Поточний стан (аудит)
1. Dev-панель зараз реалізована лише у `tabbed-window`.
2. У `main-page` є тільки права кнопка `Закрити`.
3. Dev mode уже глобально керується через `body.dev-mode`, отже новий елемент можна вмикати цим же станом.

## Вимоги до реалізації
1. Нова кнопка розгортання має бути зліва у верхній смузі `main-page`.
2. Права кнопка `Закрити` має залишитись без втрати функціоналу.
3. Функціонал панелі в `main-page` має бути ідентичний наявному:
- перемикання курсів
- перемикання тем
- активні стани кнопок
- закриття по backdrop
- закриття по `Escape`
4. Показ/приховування контролів тільки в dev mode.
5. Робота у всіх темах (`classic`, `cyberpunk`, `kawaii`) без візуальних зламів.

## Технічний план
1. Розмітка (`root/index.html`):
- додати лівий toggle-контейнер і кнопку в `training-close-bar`;
- додати для `main-page` контейнер dev-панелі + backdrop;
- додати `aria-controls` і `aria-expanded`.

2. JS (`root/index.html`, inline script):
- винести спільну логіку open/close dev-панелі в універсальні функції;
- підключити цю логіку і до існуючої панелі `tabbed-window`, і до нової панелі `main-page`;
- використати ті ж джерела даних для рендеру курсів/тем (`settingsCourseSelect`, `settingsThemeSelect`);
- синхронізувати активні кнопки в обох панелях.

3. CSS (`shared/css/app.css` + за потреби точково в темах):
- стилі лівої кнопки в `training-close-bar` (по візуалу як існуюча `header-dev-expand-btn`);
- стилі для панелі/overlay у `main-page`;
- стан `body.dev-mode` та відкриття панелі через окремий клас (без конфлікту з `header-dev-panel-open`).

4. UX/A11y:
- keyboard-focus для кнопок;
- `Esc` закриває панель;
- backdrop-клік закриває панель;
- правильний `aria-expanded`.

## Обмеження
1. Не змінювати логіку етапів, штрафів, прогресу.
2. Не змінювати тексти локалей без потреби.
3. Не ламати наявну dev-панель у `tabbed-window`.

## Критерії приймання
1. У dev mode в `main-page` зліва є кнопка розгортання.
2. Відкрита панель у `main-page` має той самий функціонал, що і поточна.
3. Перемикання курсу/теми працює коректно.
4. У всіх темах та на мобільному немає візуальної кривизни.
5. Поточна панель у `tabbed-window` працює як раніше.
